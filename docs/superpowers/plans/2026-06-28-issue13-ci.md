### Task 2: GitHub Actions CI ワークフロー作成

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Consumes: Task 1 の修正済み main.rs（CI で check が通ることの前提）
- Produces: なし

- [ ] **Step 1: `.github/workflows/ci.yml` を作成**

```yaml
name: CI

on:
  push:
    branches: [master]
  pull_request:
    branches: [master]

env:
  CARGO_TERM_COLOR: always

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: thumbv6m-none-eabi
          components: clippy

      - name: Create dummy CYW43 firmware blobs
        run: |
          mkdir -p crates/firmware/cyw43-firmware
          touch crates/firmware/cyw43-firmware/43439A0.bin
          touch crates/firmware/cyw43-firmware/43439A0_clm.bin

      - name: cargo check (smtlk-core)
        run: cargo check -p smtlk-core --locked --target x86_64-unknown-linux-gnu

      - name: cargo check (smtlk-firmware)
        run: cargo check -p smtlk-firmware --locked --target thumbv6m-none-eabi

      - name: cargo clippy (smtlk-core)
        run: cargo clippy -p smtlk-core --locked --target x86_64-unknown-linux-gnu -- -D warnings

      - name: cargo clippy (smtlk-firmware)
        run: cargo clippy -p smtlk-firmware --locked --target thumbv6m-none-eabi -- -D warnings

      - name: cargo test (smtlk-core)
        run: cargo test -p smtlk-core --locked --target x86_64-unknown-linux-gnu
```

- [ ] **Step 2: コミット**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: cargo check/clippy/test の GitHub Actions ワークフローを追加"
```
