# パーツ選定ドキュメント Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `circuit/netlist.py` の BOM を日本のネット通販（秋月電子に集約）で発注できる実商品へ写像し、購入先まとめと概算を載せた調達ドキュメント `docs/parts-selection.md` を作る。

**Architecture:** 1 本の Markdown を段階的に埋める。概要 → メイン表（コア9点＋追加5点）→ 按分計算と概算サマリ → 購入先まとめと備考、の順。価格・通販コードは Web リサーチで取得し、調査日を明記したスナップショットとして記録する。netlist.py は触らず参照のみ。

**Tech Stack:** Markdown のみ。リサーチに WebSearch / WebFetch。コード変更なし。

## Global Constraints

- 出力ファイルは単一の `docs/parts-selection.md`。netlist.py その他のソースは変更しない。
- 購入先は日本のネット通販サイトのみ。秋月電子通商（akizukidenshi.com）を主軸に集約し、秋月に無い／不適な品だけ千石電商・スイッチサイエンス等に回し、理由を明記する。
- 手配線前提でスルーホール品を優先（FET は TO-220 ロジックレベル、抵抗はリード品）。SMD は採らない。
- 数量はスマートロック 1 台分。パック品はパック価格（商品単価）と 1 台按分額の両方を載せる。
- 価格・通販コードは調査日時点のスナップショット。冒頭に調査日と価格変動・在庫の注意書きを置く。
- 仕様の正は `docs/superpowers/specs/2026-06-25-parts-selection-design.md`。対象14品目とルールはそこに従う。

### Task 2: コア部品（BOM 9 点）の秋月リサーチとメイン表記入

**Files:**

リサーチ手順（各品目共通）: WebSearch / WebFetch で秋月電子通商（akizukidenshi.com）の商品ページを引き、商品名・通販コード（例 `M-12345` / `P-00112` / `R-25102` 形式）・税込価格・入数を記録する。選定はスペックの値・型の意図を満たすこと。秋月に適品が無い場合のみ千石電商・スイッチサイエンスを当たり、店名と通販コードを残す。調査日を控える。

各品目の検索キーワードと選定基準:

- U1 Raspberry Pi Pico W: 秋月で「Raspberry Pi Pico W」を検索。無線付き（W）であること。ヘッダ未実装の素の版を基準（ヘッダ付き版があれば備考に代替として記す）。

- Q1 N-ch MOSFET（ロジックレベル）: 「IRLZ44N」を第一候補に検索。スルーホール TO-220、ロジックレベル（Vgs(th) が低く 3.3V GPIO で十分オンする）であること。IRLZ44N が無ければ同等のロジックレベル Nch TO-220（例 2SK4017 は閾値が高いので不可、代わりに RFP30N06LE 等）を探し、選んだ型番と理由を備考に残す。
- Rg 抵抗 220Ω: 「カーボン抵抗 1/4W 220Ω」。リード品。多くは 100 本入パック。入数を必ず記録。
- Rgs 抵抗 10kΩ: 「カーボン抵抗 1/4W 10kΩ」。同上。
- Rled 抵抗 330Ω: 「カーボン抵抗 1/4W 330Ω」。同上。
- D1 LED 5mm: 「5mm 赤色 LED」等。状態表示用。単品 or 小袋、入数を記録。
- SW1 タクトスイッチ: 「タクトスイッチ」。基板用、単品 or 小袋。
- C1 電解コンデンサ 470µF: 「電解コンデンサ 470µF」。耐圧は VBUS(5V) に余裕を持つ 10V 以上（16V/25V でも可）。

### Task 4: 按分額の計算と概算サマリ

**Files:**
- Modify: `docs/parts-selection.md`（メイン表の按分額列を全行埋め、概算サマリを記述）

**Interfaces:**
- Consumes: Task 2・3 で埋めた全 14 行の 商品単価・入数・1台必要数。
- Produces: 各行の `1台按分額`（= 商品単価 ÷ 入数 × 1台必要数、円未満切り上げ）と、概算サマリの 2 値（①理論按分合計 ②実支出総額）。

- [ ] **Step 1: 按分額列を全 14 行ぶん計算して埋める**

各行 `1台按分額 = ceil(商品単価 ÷ 入数 × 1台必要数)`。例: 100 本入 ¥100 の抵抗を 1 本 → ¥1。Pico W のような入数 1 の単品は商品単価そのまま。

- [ ] **Step 2: 概算サマリを書く**

`## 概算サマリ` に次の 2 値と内訳を書く。

```markdown
- ① 1台分の理論按分合計: ¥xxx（按分額列の総和。理論上 1 台が消費する分だけの価値）
- ② 実際に 1 台作るために払う総額: ¥x,xxx（パック品も丸ごと買う前提の実支出。送料別）
  - 内訳: 秋月 ¥x,xxx ＋ 他店 ¥xxx
  - 送料の目安: 秋月（注文金額により変動）、他店分は別途。発注前に各店の送料条件を確認。
- 補足: ②は初回 1 台の実支出。抵抗等のパック余剰により 2 台目以降は①に近づく。
```

- [ ] **Step 3: 確認**

確認（目視＋暗算）: 按分額列に空 `-` が無いこと。①が按分額列の総和に一致すること。②が「各商品単価 × 必要購入数」の合計に一致すること。

Run: `grep -nE '\| *- *\|' docs/parts-selection.md`
Expected: 按分額列に `-` の残りが無い（ヘッダ区切り行を除く）。
