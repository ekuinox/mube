# ユニバーサル基板 配線図ジェネレータ 設計

- 日付: 2026-07-11
- 対象: `circuit/perfboard/`（新規）、`circuit/perfboard.ts`（新規 CLI）、ビューア拡張
- 関連: [2026-07-06 ブレッドボード自動 place & route](2026-07-06-breadboard-auto-place-route-design.md)、[2026-07-05 配線図](2026-07-05-wiring-diagram-design.md)

## 目的

実機で使うユニバーサル基板（秋月 P-03229 Cタイプ・72×47mm・2.54mm ピッチ）に、
smtlk の回路を**はんだ付けして実装するための配線図**を SVG で生成する。
利用者はこの図を見ながら、どの穴に何を置き、どのピンとどのピンを配線するかを追える。

既存のブレッドボード自動配置は「列タイ・電源レール」というブレッドボード固有の導通を
前提にしており、ユニバーサル基板（格子穴が独立・配線は全てはんだ）にはそのまま使えない。
そこでユニバーサル基板専用の描画系を新設する。

## 方針

- **配置は宣言的に手置き（利用者が座標指定）／配線はツールが自動描画**。
  LED・サーボ用コネクタ等の物理的な置き場所を利用者が完全に制御できることを最優先する。
- **結線の正は `circuit/parts.ts`（PARTS / NETS）を共有**する。ERC 済みの結線を再利用し、
  配線図側で電気的な二重管理を作らない。
- Pico W は**実データシートのピンアウトに基づく 2×20 ヘッダ・フットプリント**として定義し、
  使用ピンが基板上のどの穴に来るかをツールが自動導出する（利用者は Pico ピン位置を手計算しない）。

## 座標系（盤面モデル）

実基板の実測（利用者確認済み）に合わせる。

- 2.54mm ピッチの一様な格子。**左上の穴を原点 (0,0)**、**x = 横（右方向＋、短辺 47mm≒18 穴）**、
  **y = 縦（下方向＋、長辺 72mm≒28 穴）**。穴の住所は **`[x, y]` の整数ペア**（0 始まり）。
- Pico の 20 ピン列は **y（縦・長辺）方向**に走る。短辺（x・横）は 2 ピン列の並ぶ向き。
- 盤面寸法・穴数は設定値。既定は `params.scad` の `uboard_l=72 / uboard_w=47` に整合させ、
  縦 `floor(72/2.54)=28`（y=0..27）× 横 `floor(47/2.54)=18`（x=0..17）。**実物の穴数は要最終確認**。
- 描画は実基板の印字に合わせ、**上端＝文字目盛り（x：A..）／左端＝数字目盛り（y：1..）**を振り、
  実物と目視で突き合わせられるようにする（印字の原点対応は要確認。下記「未確定」参照）。

## Pico W フットプリント（本設計の核）

Pico W は 40 ピン＝2 列×20、2.54mm ピッチ、ブレッドボード互換（全ピンが 2.54mm 格子に乗る）。
ピン番号は DIP 式（USB を上に、左列を上から 1..20、右列を下から 21..40）。

**Pico は実基板上で位置固定**：利用者の実測で **GP0（物理ピン1）＝ (3, 2)**、**GP16（物理ピン21）＝ (10, 21)**。
標準ピンアウトに当てはめると全 40 ピンの (x,y) が一意に決まる:

- 左列（ピン 1..20）：x=3、`ピン n → (3, n+1)`（上→下）
- 右列（ピン 21..40）：x=10、`ピン n → (10, 42−n)`（下→上）
- 2 ピン列の間隔 = 10−3 = **7 穴**（実測一致）。`board.ts` に **`PICO_ROW_SPAN_HOLES = 7`** と
  Pico アンカー `PICO_ANCHOR = [3, 2]` を単一の名前付き定数で明示。実物と違ってもここの修正で追従する。

本回路が使う 7 本の穴:

| 信号（parts.ts の U1 ピン） | 物理ピン | (x, y) |
| --- | --- | --- |
| VBUS (V5) | 40 | (10, 2) |
| GP18 (LED_DRV_G) | 24 | (10, 18) |
| GND（既定。他 3/8/13/18/28… も可）| 23 | (10, 19) |
| GP17 (BTN) | 22 | (10, 20) |
| GP16 (LED_DRV_R) | 21 | (10, 21) |
| GP14 (GATE_DRV) | 19 | (3, 20) |
| GP15 (SERVO_SIG) | 20 | (3, 21) |

- U1 は固定アンカーなので `layout.ts` では基準として置く（基本は動かさない）。他部品を周囲に手置きする。
- Pico は全 40 ピンが基板穴を貫通・占有する。描画では 2×20 の外形をゴースト表示し、
  使用 7 ピンを強調＋信号名ラベルを付ける。**40 穴は全て「占有済み」**として扱い、
  他部品のピンが重ならないよう検証する。

## 部品フットプリント

`circuit/perfboard/footprints.ts` に、各 ref のピンをアンカー穴（＝1 番ピン）からの
格子オフセットで定義する。既存 `breadboard/footprints.ts` のピン順・極性・カソード帯の
メタ（`polarityPin` / `stripePin`）を流用する。初期スパンの既定（実装で微調整可）:

| 部品 | 形状 | ピン | 既定スパン |
| --- | --- | --- | --- |
| M1 サーボコネクタ | 1×3 ヘッダ | SIG/VPLUS/GND | 連続 3 穴 |
| Q1 MOSFET (TO-92) | 1×3 | G/D/S | 連続 3 穴（脚を 2.54 に曲げ） |
| Rg/Rgs/Rled/Rled2 抵抗 | 軸型 2 ピン | pin1/pin2 | 3 穴スパン |
| D1 二色 LED | 1×3 | R/G/K（コモンカソード）| 連続 3 穴 |
| SW1 タクトスイッチ | 2 端子（対角）| pin1/pin2 | 2×2（対角利用と注記）|
| C1 電解 470uF | 極性 2 ピン | pin1(+)/pin2 | 2 穴、+ を明示 |
| C2 セラ 100nF | 2 ピン | pin1/pin2 | 2 穴 |
| D2 フライバック Di | 軸型 2 ピン | cathode/anode | 3 穴、カソード帯 |

各部品は **rot ∈ {0,90,180,270}** で回転。ツールがアンカー＋回転から各 `Ref.pin` の穴を算出する。

## 利用者インタフェース（手置き）

`circuit/perfboard/layout.ts`（利用者が編集する唯一のファイル）:

```ts
// at: [x, y]（左上原点・0 始まり）＝各部品の 1 番ピンの穴
export const PLACEMENT: Record<string, { at: [number, number]; rot: 0 | 90 | 180 | 270 }> = {
  U1:  { at: [3, 2],  rot: 0 },   // Pico ヘッダ（固定アンカー。基本は動かさない）
  D1:  { at: [1, 24], rot: 90 },  // 二色 LED（置きたい場所が効く）
  M1:  { at: [13, 25], rot: 0 },  // サーボコネクタ
  // … 残り部品
}
```

- **最初から成立する既定レイアウトを同梱**する（全ピン盤面内・重なり無し・全ネット配線可）。
  利用者はこれを見て住所を書き換えるだけでよい。
- ライブプレビュー: 既存のブラウザビューアで保存→再描画して置き直しを即確認する。

## 配置解決と検証（place.ts）

- `PLACEMENT` を解決し `"Ref.pin" → Hole` を得る。
- 検証（失敗は明確なエラーで停止）:
  - 全ピンが盤面内（x=0..17・y=0..27、既定値。実基板の穴数で確定）。
  - **どの 2 ピンも同じ穴を共有しない**（Pico の 40 占有穴を含む）。1 穴＝1 リード。
  - NETS の全端点がフットプリントで解決できる。
- 注記として「Pico の 2 列に挟まれた内側領域は Pico 本体の上。部品はできるだけ外側へ」を出す。

## 配線描画（wire.ts）

- 各 NET について、端点の穴同士を**点対点はんだ配線**で結ぶ。
  ネット内に k 端点あれば、最小全域木（MST）で k-1 本の配線に落とす。
- ルーティングは格子追従ではなく穴中心間の直線／簡単な L 字（実際のジャンパ線に相当）。
  オートルータは作らない（YAGNI）。
- ネットごとに配色。電源系（V5 / GND / SERVO_RTN）は専用色で強調。

## 描画（render.ts, SVG）

`breadboard/render.ts` の SVG 生成方式に倣う。描く要素:

- 盤面矩形（薄色）＋全穴（円）を 2.54mm スケールで。実基板に合わせ**上端＝文字 x:A..／左端＝数字 y:1..** の目盛り。
- Pico 2×20 ゴースト外形＋使用 7 ピンの強調と信号名。
- 各部品のフットプリント・ref/値ラベル・極性(+)・カソード帯。
- ネット配線（配色）。
- 凡例（ネット色・部品ピン凡例・組立注記）と統計（使用穴数・盤外警告）。

## CLI とビューア

- `circuit/perfboard.ts`: `bun perfboard.ts` で `build/perfboard.svg` を出力（`breadboard-auto.ts` に倣う）。
- 既存ビューア（`circuit/breadboard-serve.py` + `breadboard-viewer.html`）を最小拡張:
  serve 時に `perfboard.svg` も生成し、ビューアのプルダウンに「ユニバーサル基板」を追加する。
  盤面 SVG は自前背景・viewBox を持つので既存の pan/zoom でそのまま見られる。

## ファイル構成

```
circuit/
  perfboard/
    board.ts        # 格子モデル: 住所⇔座標, 盤面寸法, PICO_ROW_SPAN_HOLES
    pico.ts         # Pico 2×20 フットプリント + 使用ピン→穴（標準ピンアウト表）
    footprints.ts   # 部品フットプリント（アンカー基準オフセット）+ 回転
    layout.ts       # 利用者編集の既定 PLACEMENT
    place.ts        # PLACEMENT 解決 + 検証
    wire.ts         # NETS → 配線セグメント（MST 点対点）
    render.ts       # SVG 生成
    *.test.ts       # 単体テスト
  perfboard.ts      # CLI（build/perfboard.svg）
```

## テスト（bun test）

- Pico ピン→穴の写像が使用 7 ピンで正しい（GP14→物理19 等）＋ `PICO_ROW_SPAN_HOLES` の反映。
- 配置検証が盤外・穴重複を検出する。
- NETS の全端点が穴に解決し、配線が全端点を連結する（各ネットが 1 連結成分）。
- render が期待要素を含む妥当な SVG を出す。
- 既存 ERC（`parts.ts` 不変）が引き続き通る。

## スコープ外（YAGNI）

- 格子追従オートルータ（点対点で十分・現実的）。
- 表裏レイヤの厳密モデル（配線は線で描き、凡例に注記）。
- 電源バスレールの最適化（v1 は配色 MST。将来拡張余地）。
- ドラッグ GUI エディタ（テキスト設定＋ライブプレビューで代替）。

## 未確定・確認事項

- `PICO_ROW_SPAN_HOLES = 7` / `PICO_ANCHOR = [3,2]` は利用者実測に一致。念のため実物で最終確認。
- **実基板の穴数**（x 方向・y 方向の最大）。既定 18×28 は寸法からの概算。実物の目盛り最大で確定する。
- **印字目盛りの原点対応**：左上の穴 (0,0) が実基板でどの文字・数字か（例：`A1`）。
  図の軸ラベルを実物に合わせるため確認する。
- 既定 PLACEMENT の具体座標は実装時に「盤面内・重なり無し・見やすさ」を満たす形で確定する。
- GND に使う Pico 物理ピン（既定 23＝(10,19)）は配置の都合で近い GND ピンに変更可。
