# 電子部品トレイ（Pico + ユニバーサル基板）設計

## 目的

Pico W とユニバーサル基板の間（現状ピンヘッダ 8.5mm で浮いているだけ）に下支えが無く、基板がたわむ。これを解決しつつ組み立て性を上げるため、本体床に直付けしている Pico マウントを廃止し、独立した**電子部品トレイ**へ移す。トレイ上に Pico と基板を平置きで組み、最後にトレイごと本体へ M2 ネジで連結する。

## 背景（現状の作り）

- Pico は本体床から生えた 4 本のボス `pico_w_mounts()`（47×11.4 ピッチ, h=3）にネジ止め（`body.scad:37-39`）。
- ユニバーサル基板は Pico の上にピンヘッダ（`pin_header_h=8.5`）で乗るだけで下支えが無い（`layout_check.scad`）。
- ユニバーサル基板は秋月 P-03229 Cタイプ 72×47×1.6mm。四隅マウント穴 **φ3.2mm**、ピッチ長辺 66mm・短辺 41mm（データシート確定値、採寸不要）。信号穴は φ1mm/2.54 ピッチで別物。

## 設計

### 全体スタック（Z, トレイ床上面基準）

```
[uボード 72×47]  四隅φ3.2穴 → 背高ポスト4本に M2 セルフタップ（66×41 ピッチ）
   ‖ ピンヘッダ 8.5        ↑ ポスト高 12.5mm が基板下面を支える
[ Pico 51×21 ]  → 短ボス4本に M2（既存 pico_w_mounts, 47×11.4, h=3 流用）
[ トレイ床 tray_t ]  四隅 → 本体床の連結ボスに M2
[ 本体の床 wall ]
```

- ポスト高 `tray_post_h = pico_boss_h + pico_h + pin_header_h = 12.5`。基板下面がピンヘッダで決まる高さと一致させ、下から支える。
- ポスト XY はデータシートの 66×41 ピッチ（Pico 中心＝uボード中心を前提に ±33 Y × ±20.5 X）。Pico フットプリント（±10.5 X, ±25.5 Y）の外側に立つので Pico と干渉しない。
- φ3.2 穴に M2 ネジ＝片側 1.2mm のクリアランス。ピンヘッダ半田付けとの二重拘束はこの遊びが吸収するため、ネジ止めでも過拘束にならない。穴あけ加工は不要。

### 部品分割

- 新規 `scad/tray.scad`：トレイ本体（床 ＋ Pico 短ボス4 ＋ uボード背高ポスト4 ＋ 本体連結ネジ穴4）。1 部品・1 目的。
- `hardware.scad` の `pico_w_mounts()` はトレイから呼び再利用（重複を作らない）。
- トレイ四隅クーポン（`mount_coupon` / `pilot_gauge` に倣う薄型テストプリント）：短ボス＋背高ポストの高さとネジ効きを全体印刷前に試すため、`tray.scad` の一角を切り出す `part` を追加。

### パラメータ（`params.scad`）

- `layout_check.scad` の `pin_header_h=8.5` / `uboard_t=1.6` を params.scad へ昇格（トレイと共用）。
- Cタイプ実値を追加：`uboard_mount_span_l=66`（Y 長辺）/ `uboard_mount_span_w=41`（X 短辺）/ `uboard_mount_d=3.2`。
- `tray_t=2.4`（床厚, 既定は wall と同じ）/ `tray_post_h = pico_boss_h + pico_h + pin_header_h`。
- ポスト・連結ボスの下穴は実績値 `servo_screw_pilot=2.2`（A1 mini の縮み補正込み）を流用。

### 本体側の変更（`body.scad`）

- `pico_w_mounts()` の呼び出しを撤去（トレイへ移動）。
- 本体床に**トレイ連結ボス4本**（M2 セルフタップ下穴 `servo_screw_pilot`）を追加。位置はトレイ四隅ネジ穴に一致。
- `usb_z` をトレイ床厚ぶん補正：`usb_z = wall + tray_t + pico_boss_h + pico_h + usb_connector_h/2`。これを直さないと USB 穴とコネクタがずれる（地雷）。

### ビルド接続（`smartlock.scad` / `build.sh`）

- `smartlock.scad` に `use <tray.scad>`、`part=="tray"` 分岐、組み立てビュー `asm_tray`（色付き配置）、トレイクーポンの `part` 分岐を追加。
- `build.sh` の部品リストに `tray asm_tray` を追加。

## テスト

- `./test/render.sh scad/tray.scad`：トレイ単体がワーニング無しでレンダリングできる。
- `./build.sh`：body 改修込みで全部品＋トレイが通る（`WARNING:`/`ERROR:` で FAIL する仕組みが形状整合を担保）。
- Rust は不変のため host-test 不要。
- 実機（任意・推奨）：トレイ四隅クーポンを試し刷りし、短ボス・背高ポストの高さとネジ効き・φ3.2 穴の位置一致を確認してから全体を印刷。

## 高さバージョンの扱い

基板下面高さはピンヘッダで一意（8.5mm）に決まるため、`tray_post_h` 一本で管理する。実ヘッダが公称と違った場合は `-D tray_post_h=…` で刷り直す。別ファイルでの複数版量産はしない（YAGNI）。

## 未確定・前提

- Pico 中心＝uボード中心（`layout_check.scad` の配置踏襲）。実装時に uボードの XY 原点がこれと一致することを確認する。
- `tray_t=2.4` は暫定。トレイのたわみが気になれば増厚（USB 高さ補正式に自動反映される）。
- 本体連結ボスの本数・位置は四隅を基本とし、トレイ footprint が本体内寸（54×126）とペデスタルに干渉しないことを実装時にアサートで担保。
