# トレイ天面留め（本体裏留めの廃止）設計書

日付: 2026-07-15

ブランチ: `tray-body-mount-method`

## 目的

電子部品トレイの本体への固定を、いまの「本体裏（ドア面）からのネジ留め」から
「天面（内側・lid を外した側）からのネジ留め」に変える。ドアに本体を両面テープで
貼り付けたあとでも、トレイの取り付け・取り外し・メンテがドアを剥がさずにできるようにする。

## 背景（現状の問題）

- 現状、トレイはトレイ側に立てた固定ポスト（`tray_fix_d=6` × `tray_fix_h=6`）へ、
  **本体の裏面（`z=0` 側）から** M2 セルフタップをねじ込んで留める（`hardware.scad` の
  `tray_mount_cuts()`：床貫通シャンク穴＋裏面ザグリ。`tray.scad` は本番と同じ袋下穴を
  ポストへ持つ）。
- 座標の向き上、`z=0` の裏面＝サムターンのノブが通る開口がある面＝**ドアに貼り付く面**。
- したがって「ドアに本体を貼る面」と「トレイのネジを差し込む面」が同一。一度両面テープで
  貼るとトレイのネジ4本にアクセスできず、組み直しのたびに本体ごと剥がす必要が出る。

## 制約（なぜ単純な裏返しでは済まないか）

- トレイのネジを天面から締めるには、ねじ山を切る母材（ボス）が「押さえたいトレイ面より下」
  かつ「本体側」に無いといけない。
- ボディ床は `wall=2.4mm` しかなく、その下はドア面。ここを貫通させると効き代が足りず、かつ
  ドア面に穴が出る。よって従来の裏留めと同じ効き代（5mm）を、床だけで天面から得るのは不可。
- 効き代はボディ床から**上に立てたボス**で稼ぐ必要がある。トレイ床とボスは同じ (x,y) で
  同じ高さ帯を取り合うため、トレイ側はボスを避ける穴を開け、ボス上面に「耳（キャップ）」を
  載せてそこを締める構造にする。

## 設計方針

方式は「ボディにボス＋トレイ耳を上載せ」（自己タップ、追加金物ゼロ。既存の Pico/サーボ耳と
同じ流儀）を採用する。

### 1. ボディ側：床から立つ固定ボス

- `hardware.scad` の `tray_mount_cuts()`（裏ザグリ＋床貫通）を廃止し、`tray_mount_bosses()`
  （仮称）に差し替える。`body.scad` の `difference()` 内の呼び出しも union 側へ移す
  （カットではなく足す形状になるため）。
- `tray_fix_pts` の4点に、ボディ床上面（`z=wall`）から立つ円柱ボスを生成。
  - 外径：`tray_boss_d`（新パラメータ、目安 φ5。Pico の `pico_boss_d=5.0` に倣う）。
  - 高さ：`tray_boss_h`（新パラメータ、効き代 `tray_screw_grip=5` ＋底残しマージン。目安 6mm）。
  - 上面から `tray_screw_pilot=2.1` の袋下穴を `tray_screw_grip` 深さ。**ドア面（`z=0`）は
    貫通させない**（底に肉を残す）。
- ボス肉厚 assert：`tray_boss_d > tray_screw_pilot + 1.6`（Pico と同基準）。

### 2. トレイ側：ボスに被さるスリーブ＋キャップ耳

- `tray.scad` の固定ポスト（立ちポスト＋袋下穴）を廃止し、各固定点を次に置き換える。
  - トレイ床を貫通する逃げ穴（ボスが下から通る。径 = `tray_boss_d + 2*boss_fit`）。
  - その穴を囲むスリーブ壁（トレイ床から `tray_boss_h` の高さまで立てる）。
  - スリーブ上端のキャップ耳（`z = wall + tray_boss_h` 付近、`tray_screw_clear=2.4` のネジ通し穴）。
- ボスがスリーブへ下から刺さり、トレイの XY 位置決めも兼ねる。天面から M2 をキャップ耳に
  通してボスへセルフタップ → 耳がボス上面へ押し付いてトレイが固定される。
- キャップ耳の頭座グリは皿でなくパン頭のザグリ（`tray_head_d=4.2` / `tray_head_h=1.6`）を
  耳上面側へ。頭が lid と当たらないことを確認する。

### 3. 嵌合クリアランス（新パラメータ）

- ボス⇔スリーブの横方向すき間 `boss_fit`（新パラメータ）を導入する。`fit_clearance=0.4` を
  初期値の目安にしつつ、A1 mini の縮み（小径縦穴が約0.4細い）も踏まえ、**クーポン印刷で確定**する。
- スリーブ壁厚 `tray_sleeve_wt`（新パラメータ、目安 1.0〜1.2mm）。

### 4. 位置・クリアランスの再調整

- スリーブ外径（≒ `tray_boss_d + 2*boss_fit + 2*tray_sleeve_wt`、目安 φ7.5 前後）は現行ポスト
  φ6 より太る。以下の空きの範囲で `tray_fix_x_right` / `tray_fix_x_left`（必要なら `tray_fix_gap`）
  を調整し、assert を「ポスト径」から「スリーブ外径」基準に更新する。
  - 右ボス：ポケット右端 76.5 まで約1mm、+X 壁 86 まで約5.5mm（壁側に寄せる余地あり）。
  - 左ボス：-X 壁 −27 まで約4mm、Pico 側にも数 mm。
- 対象 assert：`params.scad` の 220〜225 行（右ポスト↔ポケット/壁、左ポスト↔Pico/壁、トレイ床 X）。
  スリーブ外径・耳の張り出しで再計算する。

### 5. 嵌合クーポン（先に刷って検証）

- `smartlock.scad` に `part == "tray_mount_coupon"` を追加。`tray_coupon` / `mount_coupon` と同じ
  方式で、**本物の body と tray の形をそのまま切り出す**。
  - 右固定点1組を含む小領域を、body（ボス側）と tray（スリーブ側）それぞれから intersection で
    切り出し、印刷しやすいよう X 方向に並べる（両方とも床下向き・ボス/スリーブ上向きで同一向き）。
- このクーポン1組で確認すること：
  1. ボスへの M2 セルフタップの効き（径 2.1・効き代の妥当性。既存 `tray_pilot_gauge` の実績確認込み）。
  2. ボス⇔スリーブの横嵌め（`boss_fit`：スルッと入りガタつかない値）。
  3. 締結時にキャップ耳がボス上面へ密着し、トレイ相当が浮かずクランプできるか。
- 既存 `tray_pilot_gauge.scad`：母材がトレイ→ボディに移るが下穴径・効き代仕様は据え置き。
  コメントを新構造に整合させる（数値は変更なし）。

## 段取り（2フェーズ）

- **フェーズ1（形＋クーポン）**：`params.scad` に新パラメータ（`tray_boss_d` / `tray_boss_h` /
  `boss_fit` / `tray_sleeve_wt`）を追加。`hardware.scad` / `body.scad` / `tray.scad` を上記に改修。
  `smartlock.scad` に `tray_mount_coupon` を追加。`./test/render.sh` で各パート NoError を確認し、
  クーポンだけ印刷して 1〜3 を実測。
- **フェーズ2（実測反映＋本体）**：実測で `boss_fit`（必要なら効き代・位置）を確定 → 本体へ反映 →
  `./build.sh` で STL 出力、viewer で目視、最終的に本体を刷って組み確認。

## 検証

- `./test/render.sh` で `body` / `tray` / `tray_mount_coupon` / `assembly` が Status: NoError。
- `params.scad` の assert が全て成立（スリーブ外径・耳基準に更新済み）。
- `./build.sh` で全パートが STL 出力できる。
- クーポン実機で嵌め合い・効き・クランプを確認し、実測値で params とコメントを整合。
- Rust は無変更のため host-test 不要。

## スコープ外

- lid・socket・ペデスタル・サーボまわりの形状（固定点の頭が lid に当たらない確認は除く）。
- ドア固定方法そのもの（両面テープ運用は前提として扱うが、貼り方の設計は別項）。
- ファームウェア・回路の変更。

## 受け入れ基準

- トレイの固定ネジが、本体をドアに貼ったまま天面（内側）から着脱できる。
- ドア面（`z=0`）に固定用の穴・ザグリが出ない。
- 全パートが Status: NoError でレンダリングでき、build.sh が通る。
- クーポン実測で嵌め合い・セルフタップの効き・クランプが成立し、params とコメントが実際の形と一致。
