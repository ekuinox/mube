# ブレッドボード配線図 自動 place & route 設計

## 背景・目的

`circuit/breadboard/` には、サーボ駆動サブ回路のブレッドボード配線図を生成するツールがある。だが配置（`servo-layout.ts`）とジャンパは**手作業**で、可読性を出すのに何度も手直しが要った。

本設計は、**部品の配置とジャンパ配線を自動化**する。部品ref集合を渡すと、`parts.ts` の結線を拾い、配置→配線→電気検証→SVG描画までを自動で行う汎用ツールにする。サーボ駆動でも LED/ボタンでも全体回路でも、同じ仕組みで1枚出せるようにする。

ブレッドボードで配線自動化が現実的な理由：**ジャンパ線は被覆されており交差してもショートしない**ため、「重ならない配線」は電気的制約ではなく可読性（美観）の問題に帰着する。PCB のような同一層非交差・DRC の制約が無く、探索で解ける度合いが高い。

## 目的（何を良い配置とするか）

主軸は2つ、副次が1つ：

1. **可読性**（主）：ジャンパの交差が最少・配線長が短い。組みながら目で追える。
2. **作業性**（主）：信号が左→右に流れる、電源/GND はレール、外部コネクタ（サーボ・USB）は端。人がブレッドボードを組む定石に沿う。
3. **コンパクトさ**（副・"できれば"）：使う列数を軽く抑える。

1 と 2 は両立しやすい（ネットの列スパン最小の並びは自然と信号を左→右に流す）。作業性の定石は**人手の指定ではなくアルゴリズム側で担保**する（完全自動）。

## スコープ

### 作るもの
- 汎用の自動 place & route パイプライン（部品ref集合→配置→配線→検証→SVG）。
- プリセットのサブ回路（`SERVO_DRIVE`, `LED_BUTTON`, `FULL`）＋任意 ref配列の指定。

### やめる/置き換えるもの
- 手書き `circuit/breadboard/servo-layout.ts` を廃止（サーボ図も自動生成に一本化）。
- `servo-nets.ts` を汎用 `subcircuit.ts` へ、`servo-verify.test.ts` を汎用 verify＋autolayout テストへ。

### 触らない/維持
- `circuit/breadboard/model.ts`（穴・列タイ・レール・union-find）を土台に維持。
- `render.ts` は維持（注記ボックスのみサブ回路データ駆動へ軽く一般化）。
- `circuit/parts.ts`（部品・結線の唯一の正）はそのまま入力ソース。

### 非対象（YAGNI）
- 実寸フットプリント（Pico の 2×20 全ピン等）。**使用ピンのみの簡易フットプリント**を採用。
- 人手のアンカー/ヒント（完全自動）。
- フル2D place&route、迷路（Lee/A*）配線。1D配置＋チャネル・ルータで十分。

## アーキテクチャ（パイプライン）

```
部品ref集合 → [subcircuit] ネット抽出(parts.ts)
            → [footprints] 使用ピン＆端寄りタグ
            → [place] 1D順序ソルバ → 各部品を列に割付
            → [route] チャネル・ルータ → ジャンパをトラックに割付
            → [verify] union-find で結線正しさ＆ショート無しを保証（実行時ゲート）
            → [render] SVG (build/breadboard-<name>.svg)
```

### モジュール（`circuit/breadboard/`）
- `footprints.ts`（新規）— 部品ごとの正規ピン並び＋メタ（`edgeAffinity`, 描画ヒント）。
- `subcircuit.ts`（新規）— 部品ref集合→`parts.ts` の NETS を絞ってネット抽出（端点2未満は除外）。`servo-nets.ts` の汎用版。
- `place.ts`（新規）— 1D配置ソルバ（純関数）。順序→列割付。
- `route.ts`（新規）— チャネル・ルータ（純関数）。配置＋ネット→ジャンパのトラック割付。
- `verify.ts`（新規）— union-find 照合を再利用可能な純関数に抽出（テストでも実行時ゲートでも使う）。
- `autolayout.ts`（新規）— place→route→verify を束ね、`render.ts` が食べる `{ PIN_HOLES, JUMPERS }` 形を出力。
- `breadboard-auto.ts`（新規 CLI）— サブ回路指定→autolayout→SVG 出力＋統計表示。
- 既存 `model.ts` / `render.ts` を再利用。

## 入力とデータモデル

### サブ回路指定
- 名前付き部品ref集合（プリセット＋任意ref配列）。`subcircuit.ts` が `parts.ts` NETS を選択部品で絞る（端点2未満は落とす）。

### フットプリント（簡易・使用ピンのみ）
- 部品ごとに正規ピン並び順＋メタを静的に持つ。実際に置くのは**選択サブ回路のネットに現れるピンだけ**（例：U1 は全体回路なら7ピン、サーボ駆動なら4ピン）。
- 各使用ピン＝1列、連続列・上段（row a）。k ピンなら k 列幅。
- メタ：`edgeAffinity`（"left"|"right"|none。外部コネクタを端へ）、`polarity`/`stripe` 等の描画ヒント（C1 の＋、D2 のカソード帯）。

### レール割当
- ネット名→レールのマップ（`V5→TP`, `GND→TN`、必要なら下段も）。レール対象ネットのピンは列間直ジャンパでなく**各ピン→レールの短スタブ**でルータが処理。

## 配置アルゴリズム（1D順序＋コスト）

- 部品の**左→右の並び順**を決める問題に還元。順に列を割当（部品iは幅ぶん、部品間に1列の隙間＝胴体重なり防止）。全部品は上段・ピンは row a。
- **コスト関数（最小化・重み付き合成）**：
  1. 配線長プロキシ：各**信号ネット**の列スパン `max列−min列` の総和。
  2. 交差プロキシ：信号ネットの列区間が重なるペア数（または重なり幅の和）。
  3. 端寄せペナルティ：`edgeAffinity` 付き部品が希望端から離れた距離。
  4. 総幅：使用列数（軽い重み）。
  - 重み `w1..w4` は調整可能な定数。既定は 1・2 を厚め、4 を薄く。
  - **レールネット（V5/GND）はコスト 1・2 から除外**（レールへ落ちるので横断配線を持たない）。コストは信号ネットのみ。
- **ソルバ**：
  - 部品数 小（≤8）：全順序を総当り（8!=40320）→ 大域最適。
  - 部品数 中（〜12+）：焼きなまし or 2-opt。乱数は **seed固定で決定的**。
- 出力：部品→割当列、各ピン→`(col, row a)`。

## 配線アルゴリズム（チャネル・ルータ）

- **レールネット（V5/GND）**：各ピン→そのピン列からレール（TP/TN）への短い縦スタブ。横断配線なし。
- **信号ネット**：
  - 列タイ活用：同列のピンは配線不要。
  - 列の連結：ネットの相異なる列をソートし隣接列を横セグメントで連結（チェーン、全域木でも可）。各セグメントは lane 行（b〜e）を走る。ピン(row a)への縦到達は**列タイで暗黙**（配線は描かず、列陰影で示す）。
  - トラック割付：**left-edge 法**で各セグメントを lane 行へ貪欲割当（重ならない最下トラック）。4トラック超過は**交差を許容**して載せる（被覆線＝電気的に無問題）。目標は視覚的重なり最小。
- 出力：JUMPERS（from/to hole＋net＋color）。`render.ts` がそのまま消費。

## 検証と異常処理

- `verify.ts`（union-find）を autolayout の**実行時ゲート**に。place+route 後、JUMPERS＋列タイ＋レールから連結を組み：
  1. 各対象ネットのピンが1グループ（全結線実現）
  2. 異なるネットが同グループに無い（ショート無し）
  3. 全ピンに穴割当（網羅）
  - 失敗＝配置/配線ロジックのバグ→**例外で停止**（CLI は該当ネット名を出し非ゼロ終了）。
  - 効果：出力は**常に電気的に正しいか、明示的に失敗**（黙って間違えない）。
- 交差・重なりはエラーでなく許容。**交差数・使用トラック数をログ**表示。
- ボード幅：配置が列数超過なら**自動で幅拡張**（COLS = max(30, 必要数)）。上限超の異常時のみエラー。

## テスト戦略

- `subcircuit.ts`：ref集合→正しいネット抽出。
- `place.ts`：既知小ケースでコスト/最適順序が期待どおり／列割付が重ならない／seed固定で決定的。
- `route.ts`：生成ジャンパが verify を通る／left-edge が同トラックに重なりを置かない（交差は記録）。
- `verify.ts`：わざと壊したレイアウト（ショート/未接続）を確実に捕まえる。
- `autolayout.ts`（要）：**各プリセット（SERVO_DRIVE/LED_BUTTON/FULL）で自動出力が verify を通る**。
- `render`：SVG に ref・ネット・ジャンパ・穴が入るスモーク。
- 決定性：同入力→2回で同一出力。

## 統合とCLI

- CLI `breadboard-auto.ts`：サブ回路名/ref配列→autolayout→`build/breadboard-<name>.svg`＋交差/トラック統計。
- `package.json`：`"breadboard": "bun breadboard-auto.ts <name>"` に汎用化。
- 手書き `servo-layout.ts` 廃止、`servo-nets.ts`→`subcircuit.ts`、`servo-verify.test.ts`→汎用 verify＋autolayout テスト。`model.ts`/`render.ts` 維持。

## リスク・未確定
- 交差最小化は重みチューニング次第。既定重みで各プリセットが「手結果と同等以上に交差が少ない」ことを実測で確認し調整。
- 中規模（〜12部品）の焼きなましパラメータ（反復数・冷却）は実測で決める。決定性のため seed 固定。
- `FULL`（全体回路12部品）が COLS 自動拡張で破綻なく収まるかは実装時に確認。
