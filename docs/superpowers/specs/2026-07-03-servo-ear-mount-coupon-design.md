# サーボ耳マウント再設計＋耳クーポン 設計書 (Issue #52)

日付: 2026-07-03

## 目的

SG90 サーボの耳（マウントタブ）のネジ固定を、M2 セルフタッピングが十分噛む形に直す。
あわせて、耳の位置と噛み合いを最小限のテストプリント（クーポン）で実機確認できるようにする。
ホーン嵌合を `socket_coupon` で追い込んだのと同じ進め方。

## 背景（現状の問題）

- 現在、耳ボス（外径 4.5mm・高さ 4.5mm）が `mount_plate.scad` のペデスタル天板から突き出している。
- しかし body 側の `sg90_cutout()` のタブスロット切り欠きがボスを途中で分断し、ネジが噛める部分は上側の浮いた約 1.6mm だけ。固定として成立していない。
- 制約 1: ボス外径 4.5mm はタブのネジ穴（約 2mm）を通らないため、「ボスがタブを貫通する」配置は取れない。
- 制約 2: 天板の真下（Z=38 付近）を回転するソケット上面（半径 17mm 超）が通るため、天板から下へ深く材料を伸ばせない。
- 制約 3: サーボを上げて「ボスの上にタブを載せる」と、ホーンとソケットの嵌合が外れるため不可。

## 設計方針

### 1. 突き出しボスの廃止＋天板の増厚（#52 修正）

- `mount_plate.scad` の突き出し耳ボス（`servo_boss_d` / `servo_boss_h` の円柱）を削除する。
- ペデスタル天板の厚みを `wall`(2.4mm) から専用パラメータ `servo_plate_t`（約 3.4〜3.6mm）に増やす。
  天板上面 Z=pedestal_top_z(42) は変えず、下方向に厚くする。
- タブは天板に直接載り、M2 セルフタッピングは天板に開けた下穴（`servo_screw_pilot`）に約 3mm 以上噛む。
- タブスロット切り欠きと干渉する突起が無くなるので、#52 の分断問題は消滅する。
- 下穴を貫通にするか止まり穴にするかは、手持ちの M2 ネジの長さで決める（ネジ先端が天板下面から突き出してソケット上面に触れないこと）。
  - 例: M2x6 なら タブ 2.5mm + ねじ込み 3.5mm → 貫通穴＋天板 3.6mm でほぼ面一。
  - 例: M2x5 なら 止まり穴（深さ 3.0mm・底残し 0.4mm）で足りる。

### 2. 耳クーポンパート `mount_coupon` の追加

- `smartlock.scad` に `part == "mount_coupon"` を追加。
- `socket_coupon` と同じ方式で、body() 全体を「天板付近の薄いスラブ（厚さ 4mm 程度）」で intersection して切り出す。
  コピーした別モデルではなく本物の部品の形をそのまま使うので、確認結果がそのまま本番に効く。
- 切り出し範囲: Z = pedestal_top_z − servo_plate_t 〜 pedestal_top_z + α（α は天板より上に残るリング縁など数 mm）。
  プリント用に Z 原点へ平行移動する。
- これ 1 枚で確認できること:
  - 耳ネジ穴の間隔（`servo_screw_span` = 27.6、要実測補正）が実機と合うか
  - 下穴径 `servo_screw_pilot` に M2 が適切に効くか（固すぎ/緩すぎ）
  - ネジの実効噛み合い深さ
  - シャフト穴とタブの位置関係

### 3. 実測値の反映

お兄ちゃんの実測（保証なしでOK、クーポンで検証する前提）:

- `servo_screw_span`: 耳ネジ穴の中心間距離
- タブのネジ穴径（下穴 `servo_screw_pilot` の妥当性判断に使用）
- タブ厚（現行 `servo_tab_h` = 2.5 の確認）
- 手持ち M2 ネジの長さ（貫通/止まり穴の判断に使用）

測定が外れていても、刷り直すのはクーポンだけなので低コストで追い込める。

## 検証

- params.scad に assert を追加:
  - ネジ実効噛み合い ≥ 3mm
  - 天板下面とソケット上面のクリアランス ≥ 0.5mm
- `./test/render.sh` および `./build.sh` の全パートが Status: NoError。
- `mount_coupon` を build 対象（または render.sh 確認対象）に含めるかは socket_coupon の扱いに合わせる。

## スコープ外

- ドア固定の突っ張り先・サムターン実寸（README「未確定」の別項目）。
- ファームウェア側の変更。

## 受け入れ基準（Issue #52 より）

- 実効ネジ噛み合いが M2 セルフタッピングに十分な深さ（目安 3mm 以上）。
- build.sh の全パートが Status: NoError でレンダリングできる。
- 実測値で params を更新し、コメントと実際の形が一致している。
