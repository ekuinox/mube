# スマートロック筐体 v2 設計仕様（実採寸の反映）

- 日付: 2026-06-24
- ステータス: 確定（v1 [2026-06-24-smart-lock-enclosure-design.md] の差分改訂）
- ブランチ: `refine-dimensions`
- 対象: 既存ドアのサムターンに後付けする SG90 サーボ式スマートロックの筐体（OpenSCAD）

## 0. 背景

v1 はサムターン実寸とドア固定を「未確定」として隔離していた。実測値が得られたので、ソケット・本体サイズ・固定方針を具体化する。測定は定規によるもので ±数 mm の誤差を含む前提。

## 1. 実採寸（確定値・mm）

- **サムターン（つまみ）**: 台形板。厚み 3（一定）、幅 28（根元）→ 25（先端）、ドア面からの突き出し（奥行）11。
- **座（エスカッション）**: 円形、直径 約46。つまみは座の中心から生える。回転対称。
- **クリアランス（回転中心から）**: 左 約30 でドア端/枠、下 約40 でドアノブ。右・上は比較的余裕あり。
- **回転**: 90°。縦向き=解錠、横向き=施錠。

誤差を吸収するため、上記は全て `params.scad` の変数として保持し、ハードコードしない。

## 2. スコープ（この改訂で作るもの）

1. **サムターンソケットの台形化**（実寸係合）
2. **サーボのつまみ同軸配置**と 90° 可動の機構的保証
3. **本体サイズ・向きの再設計**（左30/下40 を超えない、右・上へ伸ばす）
4. **固定（Q6）の段階対応**：今回は「フィット確認ビルド」、トルク対策は現物合わせの次回へ

非目標（変更なし）: Pico W ファームウェア、省電力、手回し後の状態再同期。

## 3. 設計詳細

### 3.1 ソケット（`socket.scad`）

- ポケットを**台形**にする。根元 28 → 先端 25、厚み 3、係合深さ 10（突き出し 11 のうち先端 1mm を逃がす）。各寸法に `fit_clearance` を加味。
- 根元 28mm の長辺がポケット壁に当たり回転トルクを伝える。台形なので先端側は自然に逃げる。
- params 追加: `knob_w_base=28`, `knob_w_top=25`, `knob_t=3`, `knob_h=11`, `knob_engage=10`。
- 旧 `knob_w`（単一幅）は `knob_w_base`/`knob_w_top` に置換する。

### 3.2 サーボ同軸・回転（`body.scad`）

- サーボ出力軸を**つまみ回転中心（=座中心）に同軸**配置。ソケットがサーボ側（ホーン/軸）とつまみを連結する。
- 機構として 90° 振れること（縦↔横）を保証する。角度→施錠/解錠の割り当てはファーム側課題（範囲外）。

### 3.3 本体サイズ・向き

- 回転中心を**左下寄りのアンカー**とし、本体は**右・上**へ伸ばす。
- params に方向クリアランスを追加: `clear_left=30`, `clear_down=40`（安全マージンを引いた値を内部寸法上限に反映）。
- v1 の中央対称レイアウト（約 62×45）は左にはみ出すため作り直す。サーボ軸位置を基準に、左方向の張り出しが `clear_left`、下方向が `clear_down` を超えないよう内部レイアウトを構成する。
- 余裕のある右・上方向に Pico W・MOSFET・LED・ボタン・USB を配置する。

### 3.4 固定（Q6）— 段階対応

**重要な設計上の事実**: 座（Ø46 の円）は回転対称なので、**位置決め（同軸合わせ）はできてもトルク反力は受けられない**。トルク反力は非対称な固定物（ドアノブ＝下、ドア端/枠＝左）からのみ取れる。

- **今回（フィット確認ビルド）**:
  - 底面に**座の位置決め座ぐり（Ø46.5 = 46 + クリアランス）**を設け、サーボをつまみと同軸に合わせる。
  - 固定は**きれいに剥がせるテープ**＋**下方向への突っ張りスタブ**（長さ 〜`clear_down` まで、ノブ手前で止める）を仮置き。
  - 目的の検証: ①ソケットがつまみを掴むか ②本体がクリアランス内に収まるか ③座に正しく乗り同軸になるか。
- **次回（トルク対策ビルド）**: 実フィット後にノブ形状（レバー/丸/ステム）を確認し、突っ張り/フォーク形状や左枠ブレースの可否を確定。`mount_plate()` モジュールの差し替えで対応する。

固定の作り込みを現物合わせに回すのは、ノブ 3D 形状が未確定で、いま作り込むと当て推量になるため。フェーズ方針（実機検証で詰める）と一致する。

## 4. ファイル変更（想定）

- `scad/params.scad`: サムターン台形寸法・係合深さ・方向クリアランス・座径を変数追加/置換。
- `scad/socket.scad`: ポケットを台形化。
- `scad/body.scad`: レイアウトを「左下アンカー＋右上展開」に再構成。
- `scad/mount_plate.scad`: 座の位置決め座ぐり＋下方向突っ張りスタブ（フィット確認版）。
- `test/*`: 各テストの assert/レンダリングを新パラメータに追従。

## 5. 検証観点（この改訂の完了条件）

- 全 `.scad` がパラメータ変更で破綻なく再生成でき、各テストが manifold STL を出す（WARNING/ERROR ゼロ）。
- ソケットのポケットが台形（28→25×3、深さ10）で、根元辺が壁に接する。
- 本体が回転中心から左 ≤ `clear_left`、下 ≤ `clear_down` に収まり、右・上へ展開している。
- 底面に Ø46.5 の座ぐりがあり、サーボ軸が座中心と同軸。
- 固定はテープ＋下方向スタブ＋座位置決め（`mount_plate()` 内に隔離、差し替え可能）。
