# 基板設計（電気設計）をコードで持つ 設計仕様

- 日付: 2026-06-24
- ステータス: 確定
- 対象: 自作スマートロックの電装を、ユニバーサル基板手配線前提で「コードで定義・検証」する
- 関連: `2026-06-24-smart-lock-enclosure-design.md`（筐体仕様）

## 1. スコープ

- **作るもの**:
  - 回路（部品・ピン・ネット）をテキストで定義する単一ソース `circuit/netlist.py`
  - 手配線ミスを防ぐ ERC ライト（結線チェッカ）
  - 生成物: 配線手順表 `build/from-to.md` と部品表 `build/bom.md`
- **作らないもの（非目標）**:
  - KiCad などの EDA・ガーバー（製造データ）。今回は**ユニバーサル基板の手配線**なので製造データは不要。
  - 自作 PCB（製造発注）・基板レイアウト図（穴単位の部品配置）。
  - 基板外形・スタンドオフ位置の筐体（OpenSCAD）への反映。今回スコープ外。
  - Pico W ファームウェア（GPIO の最終割当はファーム フェーズで確定）。

### 設計判断
- 手配線では必要なのは「結線の正解」と「from-to の手順表」であって製造データではない。よって標準ネットリスト形式や EDA への依存を持たず、依存ゼロの自前定義に倒す。
- 将来どうしても実 PCB を起こす場合は SKiDL（Python で KiCad ネットリスト生成）に昇格できる。今回は採らない。

## 2. リポジトリ構成

```
circuit/
  netlist.py     # 回路の定義＋チェッカ＋ジェネレータ（真実の単一ソース）。コミットする
build/
  from-to.md     # 生成物（配線手順表）。再生成可能なので .gitignore（build/ は無視済み）
  bom.md         # 生成物（部品表）
test/
  netlist_test   # 壊した netlist が非ゼロ終了することを確認するテスト
```

- `netlist.py` は既存 `viewer/serve.py` と同じ **PEP 723 `uv run --script`** で実行する（今回は依存ゼロの想定）。
- 流儀: 「ソースはコミット／生成物は `build/` で非コミット」を STL と揃える。

## 3. 回路（コードに落とす内容）

### 部品（BOM）

| Ref | 部品 | 値/型 | 備考 |
|-----|------|-------|------|
| U1 | Raspberry Pi Pico W | — | USB 給電・PWM・GPIO |
| M1 | SG90 サーボ | 3線(V+/GND/SIG) | サムターン駆動 |
| Q1 | N-ch MOSFET（ロジックレベル） | 例 AO3400 / IRLZ44N | サーボ GND をローサイド開閉 |
| Rg | ゲート直列抵抗 | ~220Ω | GPIO→ゲート |
| Rgs | ゲートプルダウン | ~10kΩ | 起動時/Hi-Z 中に Q1 を確実に OFF |
| Rled | LED 電流制限抵抗 | ~330Ω | |
| D1 | 状態 LED | 5mm | |
| SW1 | タクトスイッチ | — | 内部プルアップ・アクティブLow |
| C1 | バルクコンデンサ | ~470µF | サーボ突入/失速時の電圧降下による Pico リセットを防ぐ |

### ネット（結線）

- **+5V (VBUS)**: U1.VBUS — M1.V+ — C1(+)
- **GND**: U1.GND — Q1.S — C1(−) — Rgs下 — D1.K — SW1片側
- **SERVO_RTN（切替GND）**: M1.GND — Q1.D（この区間を Q1 で開閉）
- **SERVO_SIG**: U1.GP(servo) — M1.SIG
- **GATE**: U1.GP(gate) — Rg — Q1.G、Q1.G — Rgs — GND
- **LED**: U1.GP(led) — Rled — D1.A
- **BTN**: U1.GP(btn) — SW1 — GND

### 電気的判断

1. サーボ電源は Pico の **VBUS（USB 5V）** から取る。単一電源で配線が少ない。SG90 失速時 ~700mA を USB 電源/ケーブルが供給できる前提とし、**C1（~470µF）でブラウンアウト保護**を入れる。
2. 仕様どおり **Q1 でサーボ GND をローサイド開閉**。**Rgs（ゲートプルダウン）** を追加し、起動直後の GPIO Hi-Z でもサーボが暴れないようにする。
3. **GPIO 番号は仮**（servo=GP15 / gate=GP14 / led=GP16 / btn=GP17）。`netlist.py` 内で変数化し、ファーム フェーズで差し替え可能にする（未確定はコードのモジュール境界で隔離する流儀）。
4. ボタンは Pico **内部プルアップ＋アクティブLow**（外付け抵抗なし）。

### 注意（ファーム フェーズへの申し送り）
- サーボ電源カット（Q1 OFF）時に SERVO_SIG を駆動し続けると、信号ピン保護経由で微小電流が流れうる。電源カット前に PWM 出力を止めること。

## 4. ツール `netlist.py` の責務

1. **定義**: 部品（Ref→型・ピン名）、GPIO 割当（変数）、ネット（ネット名→[(Ref, ピン), ...]）をデータ構造で持つ。
2. **チェッカ（ERC ライト）**:
   - 各部品の全ピンがちょうど1ネットに属する（未接続・二重接続を検出）
   - 各ネットは2端子以上ある（孤立ネット検出）
   - 必須ネット（+5V / GND / SERVO_RTN）が存在する
   - 違反時は標準エラーへ内容を出し、**非ゼロ終了**する
3. **ジェネレータ**: `build/from-to.md` と `build/bom.md` を出力する。

## 5. 出力フォーマット

### from-to.md（配線手順表）
- ネット単位で見出しを作り、同一ネット内の端子をチェーン状の from→to ペアに展開する（半田付けの手順としてそのまま使える）。

```
## GND
- U1.GND → Q1.S
- Q1.S   → C1(-)
- C1(-)  → D1.K
- ...
```

### bom.md（部品表）
- セクション3の表（Ref / 部品 / 値 / 数量）を自動生成する。

## 6. リポジトリ統合・テスト

- **実行**: `uv run --script circuit/netlist.py`（= `./circuit/netlist.py`）で `build/` に生成。
- **build.sh**: STL 生成に続けて from-to / bom も生成する一行を追加（任意。独立コマンドのままでも可）。
- **.gitignore**: `build/` は無視済みなので生成物は自動で非コミット。`netlist.py` のみコミット。
- **テスト**: 既存 `test/` の流儀に合わせ、わざと壊した netlist（例: ピン未接続）でチェッカが非ゼロ終了することを確認する。正常な `netlist.py` がチェックを通ることも確認する。

## 7. 検証観点（完了条件）

- `uv run --script circuit/netlist.py` が依存解決込みで実行でき、`build/from-to.md` と `build/bom.md` を生成する。
- 正常な netlist でチェッカが成功（ゼロ終了）する。
- ピン未接続・孤立ネット・必須ネット欠落のいずれかを仕込むと、チェッカが非ゼロ終了する。
- GPIO 割当を変数で変更すると、from-to 表に反映される（未確定の隔離が機能している）。
