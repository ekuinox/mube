# スマートロック 設計仕様（v1：ガワ中心）

- 日付: 2026-06-24
- ステータス: 確定（Pico W ファームウェアは別フェーズ）
- 対象: 自作スマートロック（既存ドアのサムターンに後付け）

## 1. スコープ

- **対象**: 既存ドアのサムターンに後付けするスマートロック。ドア・錠前は無加工。
- **このフェーズで作るもの**:
  - 全体要件の確定
  - OpenSCAD による筐体（ガワ）モデル
- **このフェーズで作らないもの**:
  - Pico W のファームウェア（Wi-Fi API、キャリブレーション、手回し後の状態再同期など）
  - 省電力（deep sleep）運用

非目標（Non-goals）:
- シリンダー交換・デッドボルト直駆動などの方式は採らない（後付けサムターン式に限定）。
- BLE や別 Wi-Fi ハブは設けない（Pico W が直接 Wi-Fi）。

## 2. 全体アーキテクチャ

```
[スマホ/PC] --Wi-Fi--> [Pico W] --PWM--> [SG90 サーボ] --ソケット--> [サムターン]
                          |  └--GPIO--> [N-ch MOSFET] -- サーボ電源(GND側)を開閉
                          ├--GPIO--> [状態LED]
                          └--GPIO<-- [物理ボタン]
電源: USB 常時給電（省電力運用は将来課題）
```

### 操作方法
- 主操作は **Wi-Fi 経由**（スマホ/PC から Pico W へ）。
- 本体の **物理ボタン** で手動トグルも可能（設置直後のテスト・即時操作用）。
- 状態は本体 **LED** と Wi-Fi ページの双方で確認可能。

### 駆動方式の判断
- アクチュエータは **SG90 サーボ**。サムターンの回転は 90 度前後を想定し、サーボの約 180 度可動範囲で足りる。安価・制御が容易・配線が少ない。
- **N-ch MOSFET（ローサイド）でサーボ電源を開閉**する。待機時はサーボ電源をカットし、能動的な保持トルク・待機電流・ジッタ・ジー音を無効化する。
  - 効果: 常時給電でも待機消費を抑え、非常時に手回しできる。
  - 限界: ギヤ列の機械的フリクションは残るため、手回しは「ゴリッと固い」。実用上「非常時に回せればよい」レベル。
  - リレーではなく MOSFET を採る理由: 小型・無音・安価で、Pico の GPIO から直接駆動できる。
- 素の SG90 は位置フィードバックを持たない。電源カット中に人が手でサムターンを回すと Pico は施錠/解錠状態を見失う。復帰後の再ホーミングは **ファーム側の課題**（このフェーズ対象外）。

## 3. 筐体（OpenSCAD モデル）

### 構造
- **一体モノコック箱 ＋ 開閉可能な蓋**。デバッグと USB 抜き差しのため完全密閉しない。
- 物理的な部品分割（ヘッド分離など）は行わない。ネジ止めしない以上、分割の旨みが小さく、分割部がサーボ反力トルクの荷重点になるため。

### マウント面（固定インターフェース）
- 底面（ドア接触面）を **独立した OpenSCAD モジュール `mount_plate()`** として切り出す。
- v1 は「平らなテープ面」を実装する。
- 固定方法が確定（後述の積み残し Q6）したら、このモジュールだけを差し替える、または突っ張りアーム/ボルトオンのシムとして別パーツ化する。
- 設計判断: 未決の固定方法を「物理構造」ではなく「コード（モジュール境界）」で隔離し、本体設計を先に確定する。

### サムターン結合（ソケット）
- つまみ形状に合わせた **専用ソケット（凹型）**。一番フィットが良い方式を採用。
- 寸法はパラメトリック変数化（例: `knob_w`, `knob_t`, `knob_h`）。現物採寸後に数値を投入する運用。

### 開口・フィーチャ一覧
- サーボ軸の開口（軸が底面＝サムターン側を向く）
- ソケット突出口
- USB ケーブル口
- 状態 LED 穴 × 1
- 物理ボタン穴 × 1
- サーボ固定座
- Pico W 固定ボス
- MOSFET 配置スペース
- 配線取り回し用の経路

### 設計方針
- 全寸法をパラメトリックに（板厚、クリアランス、SG90 寸法、Pico W 寸法を変数化）。
- 3D プリント前提。はめあい/公差調整用の変数と `$fn` を用意する。

## 4. 内部レイアウト（ブロックレベル）

- サーボは軸が底面（ドア側＝サムターン側）を向くよう配置。
- Pico W・MOSFET・配線は蓋側コンパートメントに収める。
- USB はケーブル口から外へ引き出す。
- LED とボタンは前面（人が見える/押せる面）に配置。

## 5. 部品リスト（BOM 概略）

- Raspberry Pi Pico W × 1
- SG90 サーボ × 1
- N-ch MOSFET（ローサイドスイッチ用）× 1 ＋ ゲート抵抗
- 状態表示 LED × 1 ＋ 電流制限抵抗
- 物理ボタン（タクトスイッチ）× 1
- USB 給電（モバイルバッテリー/USB 電源）
- 3D プリント筐体（本体・蓋・マウント面・サムターンソケット）
- 固定用テープ（v1。種類は Q6 確定後）

## 6. 積み残し（未決事項）

- **Q6: 固定の突っ張り先 / 固定方式の詳細**（現物確認待ち）。サーボ反力トルクをテープに負わせないため、近接する固定部品（サムターン台座プレート、レバーハンドル、ドア枠など）への突っ張りを検討。`mount_plate()` で隔離済み。
- **サムターン実寸**（採寸待ち）。ソケット寸法変数で隔離済み。
- **省電力運用**（将来課題）。Wi-Fi 常時接続と電池運用は相性が悪いため、必要なら deep sleep 設計や通信方式の見直しが要る。
- **手回し後の状態再同期**（ファーム側課題）。
- **サーボトルクの実機検証**: SG90 のトルクが実際のサムターンを回せるか、実機で確認する。

## 7. 検証観点（このフェーズの完了条件）

- OpenSCAD モデルがパラメータ変更で破綻なく再生成できる。
- 主要フィーチャ（サーボ座、Pico ボス、各開口、ソケット）が干渉なく配置される。
- `mount_plate()` とソケット寸法が、本体を再設計せずに差し替え可能な独立モジュールになっている。
- 3D プリント可能な形状（オーバーハング・最小肉厚）になっている。
