# 筐体拡大＋ブレッドボードトレイ 新設（workstream B）

- 日付: 2026-07-12
- 想定ブランチ: **新ブランチ**（例 `feat/breadboard-tray` / `feat/enclosure-breadboard`）＋**新 issue**（workstream A の #70 とは別 PR）
- 派生元: `origin/master`（eb407af / #68）想定。A（ビューア移植）とは独立。
- 位置づけ: ブレッドボード実装への物理側対応。電子部品トレイを「Pico＋スタック基板」から「Pico＋ブレッドボードを横並び」に作り替え、収まるよう筐体を拡大する。

## 背景・前提の確定

方針転換で電子部品はブレッドボード（実測 **85.5 × 54.5mm**、ハーフ 400穴相当）に載る。現筐体の電子部品ベイ（+Y、内寸 X=54 / +Y 使用可 ≈74.6mm）は旧スタック基板 47×72 用で、**ブレッドボード単体でも収まらない**（54.5>54 / 85.5>74.6）。→ 筐体拡大が必要（ユーザー決定）。

### 座標・向き（repo 採寸の正で確定）

- 原点 = サムターン／サーボ軸（ロゼット中心）。
- **−X = 左（ドア枠・端）** 上限 `clear_left = 50`、**−Y = 下（ドアハンドル/レバー）** 上限 `clear_down = 65` … **硬い制約（動かさない）**。
- **+X = 右**、**+Y = 上** … クリアランス制約なし＝**拡大はこの2方向に逃がす**。
- ユーザー確認: ドア正面ビューでハンドルは「下」＝ −Y。コードの想定通り。

## レイアウト（決定）

真上ビュー（+Y=上, +X=右）:

```
   ┌──────────────────────────────┐ ← ext_up ≈ 118 (+Y へ拡大)
   │  ┌──────┐   ┌────────────┐   │
   │  │ Pico │   │ ブレッド    │   │
   │  │51×21 │   │ ボード      │   │
   │  │ x=0  │   │ 85.5×54.5  │   │
   │  └──────┘   └────────────┘   │
   │      ╭──────╮                │
   │      │ペデスタ│  ← サーボ台座   │
   │      ╰──╳───╯  (r=25.4)      │
   └──────────────────────────────┘ ← ext_down = 26 (不変)
   -X(左 27 不変)          +X(右 ≈ 72 へ拡大)
```

- **Pico は据え置き**: `pico_x = 0`（左右中央）・`pico_y ≈ 63.4`（+Y、ペデスタル真上）・長軸 Y・USB は +Y 壁から（変更最小）。
- **ブレッドボードは Pico の右（+X）に横付け**、**長辺 85.5 を Y 沿い（縦長）**。
- Pico↔BB の隙間 ≈ 4mm（ジャンパ用）。

### 概算寸法（実装時に params で確定・assert で担保）

- Pico 右端 x = +10.5。BB 左端 x ≈ +14.5、BB 右端 x ≈ +69。
- BB 下端 y ≈ 28（ペデスタル外径 25.4 をクリア）、BB 上端 y ≈ 113.5。
- → `ext_right: 27 → ≈72`、`ext_up: 100 → ≈118`。`ext_left=27` / `ext_down=26` は不変。
- 筐体外形 **≈ 103 × 149mm**。
- クリアランス assert: 実現 −X extent（`body_l/2 - center_x`）が `clear_left=50` 以下、実現 −Y extent が `clear_down=65` 以下を維持（+X/+Y 拡大なので自然に満たすが assert で確認）。

## 固定方式（決定）

- **ブレッドボード → 浅い囲い壁ポケット**: BB 外形＋クリアランス（例 0.5mm）に合わせた低い壁で囲って落とし込み。位置決め確実・印刷容易。裏の粘着フォームで完全固定も可（任意）。
- **Pico → 位置決めピン継続＋コーナー爪**: 現行の位置決めピン（φ1.9 を Pico φ2.1 穴へ）を残し、Pico の端を上から押さえる**スナップ爪を 1〜2 個**追加。ジャンパを差しても浮かない。ネジは使わない（穴位置がピンヘッダのプラに当たるため元々不採用）。
- **トレイ → 本体**: 従来通り**底面から M2 セルフタップ**（本体床にザグリ）。トレイ四隅付近の固定ポストへ。旧設計の「基板支持ポスト＝トレイ固定ポスト兼用」は無くなるので、固定専用ポストを配置し直す。
