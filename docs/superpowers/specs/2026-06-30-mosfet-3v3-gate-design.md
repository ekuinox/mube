# MOSFET の 3.3V ゲート駆動: 現状維持＋根拠明記（Issue #24）

## 背景

Q1 = `IRLB3813PBF`（秋月 I-06270, N-ch 30V/260A）は、サーボの GND リターン（`SERVO_RTN`）を切るローサイドスイッチ。ゲートは Pico W の GP14（3.3V ロジック）から `Rg`(220Ω) 経由で駆動し、`Rgs`(10k) でプルダウンする。駆動はワンショット（給電時のみ ON、約 500ms）。

懸念（Issue #24）: IRLB3813PBF の Vgs(th) は最大 2.35V で、3.3V GPIO で閾値は超えるが、Rds(on) が規定される Vgs（4.5V / 10V）には届かない。3.3V 駆動では Rds(on) が規定点より増え、サーボ GND リターン経路に余計な電圧降下が生じ、サーボへの実効電圧が下がりうる。

`#17` で本命の IRLZ44N が秋月在庫なしのため IRLB3813PBF を確定した経緯があり、調達は「秋月電子に全 16 品集約」「スルーホール・手配線」が強い制約。親 Issue: #12 / 対象 Issue: #24。

## 目的

回路を変更せず IRLB3813PBF を維持し、3.3V ゲート駆動で実用上問題ないことの定量的な根拠と、実機ブリングアップ時の計測手順・受け入れ基準を `docs/parts-selection.md` に明記して Issue #24 をクローズする。

## 方針の選定

検討した 3 案:

1. **現状維持＋根拠明記（採用）**: 回路変更なし。定量根拠と計測手順を文書化。
2. ロジックレベル品へ変更: Vgs(th) が低いスルーホール・秋月入手可の N-FET を再選定し差し替え。入手可否調査が前提で、netlist / BOM / SCAD キープアウトに波及。
3. 5V ゲート駆動を追加: 小信号トランジスタ等でゲートを 5V までスイングし Vgs≥4.5V を保証。部品追加・回路複雑化。500ms ワンショットには過剰。

採用理由: IRLB3813PBF は 260A 級で Rds(on) が極小（後述）。SG90 の負荷では 3.3V 駆動でも電圧降下が無視できる見込みで、最も低コスト・低リスク。実機計測で万一基準を超えたら案2/案3へエスカレーションする余地を残す。

## 定量的な根拠（データシート実値）

出典: Infineon IRLB3813 データシート（Vgs(th): min 1.35V / typ 1.9V / max 2.35V、Rds(on) @ Vgs=4.5V = 2.6mΩ max、@ Vgs=10V = 1.95mΩ max）。Rds(on) が Vgs=4.5V で規定されている点が、本品がロジックレベル MOSFET であることの裏付け。

- **十分にオンする**: Vgs=3.3V のゲートオーバードライブは、typ 閾値 1.9V に対し約 1.4V、最悪の閾値 max 2.35V でも約 0.95V。3.3V でエンハンスメント領域に入る。
- **電圧降下は無視できる**: 3.3V は Rds(on) 規定点（4.5V）をわずかに下回るのみ。保守的に Rds(on) を高々数十 mΩ と見積もっても、SG90 の起動・失速ピーク（〜1A 程度）での降下は最悪約 50mV、実際は 10mV 前後。5V 供給に対して無視できる。
- **発熱も無視できる**: ワンショット 500ms のため、ピーク電力は P = I²·Rds(on) ≒ 1²×0.05 = 50mW 程度。連続通電でないため発熱は問題にならない。

## 実機ブリングアップ時の計測手順と受け入れ基準

文書化して受け入れ条件として残す（Issue 案2「実機計測」の正式化）:

1. サーボを動作させ、可能なら失速させてピーク電流を引かせる。
2. その状態で Q1 のドレイン-ソース間電圧 Vds を計測する。
3. **受け入れ基準: ピーク負荷で Vds ≤ 0.2V**。これにより 5V 供給から実効サーボ電圧 ≥ 4.8V を確保し、SG90 定格（4.8〜6V）内に収まる。
4. 基準超過時のエスカレーション: 案2（ロジックレベル品へ差し替え）または案3（5V ゲート駆動の追加）を実施する。
