# 実装ガイド（配線図）生成 設計

## 背景・目的

回路の正はすでに tscircuit（`circuit/index.tsx` が本番配線、`netlist.tsx` + `erc.ts` で circuit JSON 化・ERC）にある。
だが今あるのは「回路図（schematic）＋導通/ショート ERC」までで、**ユニバーサル基板の上で部品をどこに置いてどう手配線するか、の絵（実装ガイド＝配線図）は無い**。

本設計は、はんだ組み立て時に机で見ながら組める **実装ガイド（配線図）** を生成できるようにする。
アプローチを 1 つに賭けず、**軽い方（tscircuit の PCB ビュー）を先に出して評価し、足りなければ自作 SVG を追加する** 段階方式を採る。

## 現状の要点（前提）

- 回路の正: `circuit/index.tsx`（部品・ピン・ネット=trace・GPIO 割当）。`schX/schY` で回路図レイアウトを持つが **物理配置（pcbX/pcbY）は無い**。
- 部品のフットプリントは `0603 / 1206 / sod123` などの **SMD 仮フットプリント**。回路図と ERC を通すための代用で、実物の寸法・ピン間隔とは合っていない。
- 実際に組むのは全部スルーホール部品（SG90 コネクタ、TO-220 の MOSFET、リード付き抵抗、電解コン、タクトスイッチ、2 色 LED、Pico W ヘッダ）。ピン間隔 2.54mm でユニバーサル基板の穴ピッチに乗る。
- 実装ガイドを物理的に正しくするには、この仮フットを **実物スルーホールのフットプリント＋物理配置** に置き換える下ごしらえが要る。これはアプローチ①②共通の前提。

## スコープ

### 作るもの

- 物理版のエントリと配置データ（下ごしらえ）。
  - `circuit/board.tsx`: 物理版の board（実物スルーホールのフットプリント＋`pcbX/pcbY`）。
  - `circuit/placement.ts`: 各部品 → 基板グリッド座標＋向き＋実物フットプリント名 をまとめた手保守データ。
  - 部品＋結線（ネット）の定義は `index.tsx` から **共有部品として切り出し**、`index.tsx`（回路図）と `board.tsx`（物理）の両方が使う。結線は 1 ソースのまま。
- アプローチ②: tscircuit の PCB ビューを `build/wiring-pcb.svg` として出力するスクリプト。
- （②で不十分だった場合のみ）アプローチ①: 2.54mm 穴グリッドを描いた自作 SVG を `build/wiring.svg` として出力するスクリプト。

### 触らない / 変えないもの

- `circuit/index.tsx` の SMD 仮フットプリントと `schX/schY`（回路図の見た目）。今回は原則触らず、共有分の切り出しだけ行う。
- `erc.ts` / `check.ts` の ERC ロジック。結線は共通なので physical でも同じ ERC が通る想定。
- ファーム・SCAD・viewer。

### 将来（本設計の外・前提だけ記録）

- スルーホールを回路の正に昇格させる方針。移行時は物理版を既定にできるよう、今から分離しておく。本設計では昇格そのものは行わない。

## モジュール構成

狙いは「**結線（ネット）は 1 ソースのまま、物理フットプリント＋配置だけを足す**」こと。ネットの二重管理はしない。

- `circuit/index.tsx` … 回路図の正（schematic entry、現状維持）。
- `circuit/board.tsx`（新規） … 物理版エントリ。共有結線を使い、フットプリントを実物スルーホールへ、配置を `placement.ts` から与える。
- `circuit/placement.ts`（新規） … 物理の手保守データ（座標・向き・実物フットプリント名）を隔離。物理で「手で決める部分」はここだけ。
- 共有結線 … 部品＋trace の定義を小さな共有部品に切り出し、`index.tsx` と `board.tsx` から使う。`index.tsx` からその共有分を抜く軽い整理が入る。

（別ボードを丸ごと複製する案は、部品リストが二重化するため採らない。）

## 対象基板・実物フットプリント

- 対象ユニバーサル基板: **秋月 C タイプ（72 × 47mm、2.54mm ピッチ）**。①のグリッドも②の board 外形もこれに合わせる。
- 実物フットプリント割り当て（目安）: 抵抗＝軸形リード、Q1＝TO-220（3 ピン）、C1＝ラジアル電解、C2＝2.54 ラジアル、D1＝3 ピン 2.54、SW1＝タクト、M1＝3 ピンヘッダ、D2＝軸形（1N5819 実物）、U1＝Pico W ヘッダ（使用 7 ピン）。
- ピン間隔・寸法は **推測しない**。`docs/parts-selection.md` とデータシートから確定する。出典で確定できない値は実装中にユーザー確認を挟む。

## 出力するもの（②と①が実際に出す絵）

### ② tscircuit の PCB ビュー（`build/wiring-pcb.svg`）

- 出るもの: 実物フットプリントを `pcbX/pcbY` で並べた基板の絵＋ピン間の接続線。`routingDisabled` のままなら **ラッツネスト（未配線の細い直線＝「このパッドとここを繋げ」の線）** が出る。配置と、どこ同士を繋ぐかが分かる。
- 限界: これは **PCB（プリント基板）の絵**であり、**ユニバーサル基板の 2.54mm 穴グリッドは描かれない**。「どのマスの穴に挿すか」を穴目盛りで数える用途には弱い。ラッツネストは直線で、実際の線の這わせ方は自分で判断する。
- メンテ: `pcbX/pcbY` を足すだけ。tscircuit 標準機能で軽い。← 期待している「メンテが楽」はここ。

### ① 自作 SVG 生成（`build/wiring.svg`、②で不十分な場合のみ）

- 出るもの: **2.54mm の穴グリッドを実際に描いた**基板の絵に、部品ブロックを配置し、ネットごとに色分けした点対点の配線を引く。「この穴からこの穴へジャンパ」を目で追える、はんだ組み立て特化の図。circuit JSON から結線は自動追従。
- 限界: 自作なので描画コードを書く／保守する必要がある。見た目のロジックは手作り。
- メンテ: 重い。ただし穴グリッド＋配線可視化は①でしか出せない。

## ツール・成果物

- `circuit/` の bun ツールチェーンに乗せる。生成コマンドは `circuit/package.json` に追加（例 `bun run wiring`）。
- 出力 `build/wiring-pcb.svg`（②）、必要になれば `build/wiring.svg`（①）。**どちらも `build/` 配下＝再生成できる派生物。コミットしない**（.gitignore 済み）。
- ERC は従来どおり通す。物理版でも同じ ERC が通ることを確認する。

## 段取り（無駄なく）

1. 下ごしらえ: 共有結線の切り出し＋`board.tsx`＋`placement.ts`＋実物フット＋物理配置。physical でも ERC が通ることを確認。
2. ②を出力: `build/wiring-pcb.svg` を生成し、「これで組めるか」を評価。
3. 判定: ②で十分ならそこで完了（メンテが楽な②を採用）。穴が数えられず組みにくければ①（穴グリッド付き SVG）を追加実装して見比べる。

## 「②で十分か」の評価軸（先に固定）

以下を満たし「机で見ながら組めそう」なら②合格とする。①へ進むのは満たせないとき。

- 各部品の位置と向きが判別できる。
- どのピン同士を繋ぐか（ラッツネスト）が追える。
- 配置が 72 × 47mm に収まる。

## リスク・未確定

- 実物フットプリントのピン間隔・外形寸法（データシート／`docs/parts-selection.md` で確定。不明分はユーザー確認）。
- U1（Pico W）を「使用 7 ピンのヘッダ」で表すか「実物の 40 ピンモジュール外形」で表すか。実装ガイドとしての見やすさで実装時に決める。
- tscircuit の PCB SVG 出力で、ラッツネストが期待どおり描画されるか（`routingDisabled` 時の挙動）は実装時に確認。期待と違えば①へ倒す判断材料にする。
