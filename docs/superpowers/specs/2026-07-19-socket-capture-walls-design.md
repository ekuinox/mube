# ソケット v2: キャプチャ壁＋マウントパッド Z 補正 設計

Issue #77 の実機調査から出発した設計。結論として「throw 不足」は誤診で、根本原因は
**ホーンバー⇔ソケットの軸方向の掛かり不足**だった。本スペックはその対策を定める。

## 実機で確定した事実（2026-07-19）

- SG90（現個体）は `SERVO_MIN_US=1000 / SERVO_MAX_US=2000 / LOCK_DEG=180 / UNLOCK_DEG=0`
  の設定で、施錠⇔解錠を繰り返し完遂できる。**throw もトルクも足りている**。
  ただし台座を手でドア側へ押し付けた場合に限る。
- 押し付けない場合、施錠負荷でホーンバーがポケット（深さ 2.0mm・掛かり 1.7mm）から
  乗り上げて外れ、ガタガタと歯飛び様の音を出して施錠に失敗する。
- 低パルス側の唸り（~1250µs 以下で内部メカ端）は既知のまま。UNLOCK=0(1000µs) は
  メカ端に当たった状態だが、給電ゲートで整定後に電源が切れるため実用上は許容。
- サムターンは 90° バネ戻りタイプ（途中保持なし）。
- 施錠・解錠の方向は当初の想定と逆（LOCK=高パルス側）。

この結果、サーボ交換（DS3218 等）・ギア機構・ホーンレス直嵌合（スプライン歯形）は
**すべて棚上げ**。SG90 続投で結合部の掛かりを設計で確保する。

## 要件

1. ホーンバーの軸方向の掛かりを、取付誤差に対して鈍感にする。
   目標: 軸方向 ±2mm のズレでもバーがポケットから外れない。
2. プレートのドア固定は面ファスナー（マジックテープ類・呼び厚 6mm）を想定する。
   厚みは params で明示し、Z スタック（ペデスタル高さ）で補正する。
3. トルク伝達は従来どおりバーのキー嵌合。追加要素は伝達に介在しない。
4. 印刷は A1 mini（0.4 ノズル）。細部はクーポン実測の知見に従う。

## 設計

### 1. `mount_pad_t`（面ファスナー厚）の Z スタック組み込み

- `params.scad` に `mount_pad_t = 6` を新設（呼び厚。実測で確定するまで暫定）。
- ドア面基準の量（サムターン突き出し `knob_h`）とプレート基準の量の間に
  このオフセットを入れる。具体的には
  `pedestal_top_z = (knob_h - knob_engage - mount_pad_t) + socket_oh + horn_h`。
  ペデスタルが 6mm 低くなり、ホーンがソケット設計位置へ戻る（主補正）。
- ソケット自体はサムターン（ドア）基準なので寸法不変。

### 2. キャプチャ壁（ソケット、鈍感化）

- ホーンポケットの両端付近、バーの両脇（Y 方向）に高さ `sock_wall_h = 5` の壁を
  立てる（現行のポケット深さ 2.0 の外側に、サーボ側へ延長）。
  バーが 3mm 浮いても壁の中に留まる。
- 壁の上端内側は 45° の外開きファンネルで、組み付け時の誘い込みを兼ねる。
- 中央はギアヘッドのドームが入るため開けておく。壁の内端 X 位置
  `sock_wall_x0` はドーム径（`servo_dome_d`、要実測・暫定 12）＋余裕から決める。
- 壁上端はペデスタル天板下面（ソケット上面から `horn_h - servo_plate_t` = 6.9）
  より 0.5mm 以上低いこと（assert で担保）。

### 3. スナップ爪（オプション・`socket_claws` パラメータ）

- クーポン v3（くさび予圧・返し 1.1・垂直先端面 0.6・梁 ~8mm）の形状を、
  バー両端の爪として統合できるようにする。既定は無効（壁のみ）。
- 爪の狙いは保持ではなく**連結**: プレートが面ファスナーの伸びで浮いた時、
  ソケットがホーンに追従してノブ嵌合（10mm）側でスライドし、バー掛かりを守る。
  副次的に、ペデスタル＋サーボ＋ホーン＋ソケットを 1 ユニットとして
  着脱できるようになる（ドア前の作業は「ノブに被せて押すだけ」）。
- 予圧量（0.15 / 0.30）はクーポン v3 の実機結果で確定する。

### 4. 運用・組立

- 面ファスナーはプレート四隅（軸から遠い位置）に貼り、施錠反力による
  プレート回転を抑える。初回施錠時に回転・浮きを目視確認する。
- 座（ロゼット）による横位置決めは、パッド 6mm を挟むと届かない可能性がある。
  ロゼットのドアからの突き出し量を実測し、不足するならプレート開口に
  嵌める位置決めリング（別部品）を追って検討する（本スペックの範囲外）。
