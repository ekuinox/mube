# オープンベースプレート化＋ペデスタルのボルトオン分離 設計書

日付: 2026-07-16

ブランチ: `tray-body-mount-method`（トレイ天面留め PR #74 の続き。同系統の「貼った後でも交換できる」改修）

## 目的

1. ロゼット台座（ペデスタル）を本体から分離し、天面（内側）から M2 で着脱できるボルトオン部品にする。ドア採寸系の未確定値（`knob_h` / `rosette_d` 等）が変わっても、小さいペデスタルだけ刷り直せば済むようにする。
2. 外周の壁と蓋を全廃し、本体を「ドアに貼る1枚のベースプレート」に集約する。プリント量・時間を大幅に減らし、全部品へのアクセスを常時確保する。

## 背景

- 現行 `mount_plate()` は本体床とペデスタル（円筒 48.4mm＋サーボ天板）が一体成形で、`body()` に union されている。ドア実測が変わるたびに背の高い本体を丸ごと刷り直しになる。トレイ天面留め（PR #74）と同じ課題。
- 壁・蓋の主な役目は床の剛性化と部品保護だが、本機は床面を両面テープでドアに貼るため、使用中の剛性はドアが肩代わりする。ペデスタルのサーボ反力トルクの経路も「ペデスタル→床→テープ→ドア」で、外周壁は寄与していない。
- SG90 反力トルクは高々 0.2〜0.5 N·m 程度。回り止め形状＋M2 数本で受けられる規模で、ネジ自体にトルクを負わせない設計にすれば成立する。

## 設計方針

### 1. ベースプレート（旧 body を置き換え）

- 外形は現行床と同じ 150.8×115.8×2.4mm、角R2。ドア面（z=0）はフラット維持。トレイ同様、ドア面に穴・ザグリを出さない。
- 中央ロゼット開口 φ`rosette_d + fit_clearance` は不変。
- 上面リブ（案A）: 外周一周＋横桟2〜3本、高さ4mm・幅2mm。手持ち時の剛性と印刷反り対策。ペデスタルポケット・各ボスの根本に接続して補強を兼ねる。トレイ床の下は避けて配置（トレイは床にベタ載せのままにする）。
- ペデスタル取付部（案1: 受けポケット方式）:
  - 床上面に環状の受けポケット。内径 = ペデスタル底フランジ外径 + `pedestal_fit`（嵌め合い、クーポン実測で確定）。深さ = フランジ厚（面一に沈む）。
  - ポケット壁に回り止めノッチ 2 箇所（対角）。ペデスタル側タブと噛んでサーボ反力トルクを受ける。ネジはトルクを受けず抜け止めに徹する。
  - M2 ボス 3 本（`tray_boss_d` / `tray_boss_h` と同仕様）。位置はポケット外周まわりに等配。
- トレイ取付部: 既存 4 ボス、位置・仕様そのまま。
- 壁・蓋・USB 壁開口は全廃。USB ケーブルはオープンなので開口不要。

### 2. ペデスタル（新規の印刷部品）

- 現行の円筒（壁厚 `pedestal_wall_t=2.5`）＋サーボ天板（`servo_plate_t=3.5`、下穴・シャフト穴は現行のまま）に以下を追加:
  - 底フランジ: 筒外径から張り出す円盤。厚さ = ポケット深さ（面一）。外径がポケット内径と嵌合して軸センタリングを担う。
  - 回り止めタブ 2 箇所: フランジ外周から張り出し、ポケットのノッチに噛む。
  - 固定スリーブ 3 個: トレイと同じ構造（`tray_cap_t=3.8`、ボア全径起点の自己サポートファンネル穴、頭ザグリ）。プレートのボスに被さり天面から M2 セルフタップ。
- `pedestal_top_z = 48.4` は不変（フランジがポケットへ沈むため）。サーボ/ホーン/ソケットの噛み合いチェーン（既存 asserts）は無傷で維持する。
- 印刷向きはフランジ底を下（現行 body と同じ向き）。天板まわりの印刷条件は従来と同一。

### 3. 消すもの

- `scad/lid.scad`（蓋）とそのビルド・パート参照（`smartlock.scad` の `lid` / `asm_lid`、`build.sh` の lid）。
- body の外周壁シェル、USB 壁開口（`usb_cutout()` 呼び出しと壁開口系パラメータ）。
- 壁・蓋前提のパラメータ / asserts（`body_h` / `inner_h` / `lid_*` / USB 開口系）。整理時に他所（トレイ等）から参照されていないことを確認して消す。

### 4. 嵌合クーポン（先に刷って検証）

- トレイ天面留めと同じ進め方。`smartlock.scad` に「ポケット＋ノッチ＋ボス1本（プレート側）」と「フランジ＋タブ＋スリーブ1個（ペデスタル側）」を本物の形状から切り出して並べるクーポン part を追加。
- クーポンで確認: フランジ⇔ポケットの嵌め合い（`pedestal_fit`）、ノッチ⇔タブの噛み（ガタ）、M2 の効き、面一に沈むか。
- ファンネル穴・`boss_fit=0.4`・下穴 2.1 はトレイで実機検証済みの値を流用する（同条件）。

## 段取り（2フェーズ）

- フェーズ1（形＋クーポン）: params 追加（`pedestal_fit` / フランジ・ポケット・ノッチ寸法）、mount_plate 分割（プレート側とペデスタル側）、lid・壁の削除、クーポン part 追加。render/build 全パート NoError。
- フェーズ2（物理）: クーポン印刷 → `pedestal_fit` とノッチ噛みを実測確定 → プレート＋ペデスタル＋トレイを印刷して通し組み確認。

## 検証

- `./test/render.sh` / `./build.sh` 全パート Status: NoError。
- サーボ噛み合いチェーンの既存 asserts（`pedestal_top_z` 派生）が全て成立し続けること。
- ドア面（z=0）に穴・ザグリなし（トレイ・ペデスタル共通の原則）。
- クーポン実機で嵌合・回り止め・面一・ネジ効きを確認し、実測値で params とコメントを整合。

## スコープ外

- socket・ホーン・サーボまわりの形状変更。
- トレイの形状変更（取り付け先がプレートになるだけで、トレイ自体は不変）。
- ドア採寸系の値の確定（本改修はそれを安く吸収するための構造変更）。
- ファームウェア・回路。

## 受け入れ基準

- ペデスタルが、本体をドアに貼ったまま天面（内側）から着脱できる。
- ペデスタル交換時の刷り直しがペデスタル単品で済む。
- lid / 壁なしで全部品（トレイ・ペデスタル・ソケット・サーボ・配線）へ常時アクセスできる。
- `pedestal_top_z=48.4` が維持され、サーボ/ホーン/ソケットの噛み合いが従来のまま成立する。
- 全パート render / build NoError、既存＋新規 asserts 成立。
